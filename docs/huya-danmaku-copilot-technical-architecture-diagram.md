## 核心架构分层详解

### 1. 输入层 (Input Layer)
* **WebSocket 实时抓取**：基于 `huya-danmu` 与底层 `local_huya_danmu` 建立与虎牙服务器的长连接，抓取原始消息包并过滤出 `chat`、`gift` 等信令。
* **房间资料拉取**：调用虎牙接口解析房间的主播名、标题、分类，作为 AI 联网搜索的前置上下文。
* **标注与私域输入**：支持主播及房管手动补充梗语义释义，实时沉淀进知识库。

### 2. 核心分析层 (Core Analysis Layer)
* **清洗与归一化**：去重、过滤刷屏符号、提取 2-4 字切片关键字。
* **滑动时间窗口**：维护 10s、30s、60s 内存多级队列。
* **规则算法引擎 (`analysis.js`)**：
  * **热词/爆发检测**：计算暴增增长率与热度分数（$Heat$）。
  * **情绪与节奏感知**：判定“欢乐”、“调侃”、“焦躁”等状态。
  * **风险评分评估**：基于人身攻击与恶意复读占比得出 $Risk$ 分数。
  * **电竞阵营趋势**：检测两队（如 BLG vs T1）加油弹幕的比例倾向。

### 3. AI 与知识层 (AI & Knowledge Layer)
* **联网搜索 (Web Search)**：基于百炼 API 的 `web_search` 插件，实时检索最新出炉的网络热梗起源。
* **RAG 检索增强**：关联本地热梗库、电竞队伍别名词典和历史话术模板，加速梗匹配，规避大模型延迟。
* **大语言模型 (Qwen/Gemini)**：解析特征参数与上下文，输出高情商、短平快的主播口播和场控候选弹幕。
* **安全二次审核**：校验候选弹幕是否含违规词、是否具备导向正确性，实行高风险一律不自动发策略。

### 4. 场控动作层 (Action Layer)
* **主播提词**：接梗、爆了、降温三类话术的实时呈现。
* **互动/投票生成**：基于氛围自动生成轻互动玩法。
* **切片打点**：捕获梗爆发初期的 Source Timestamp，记录短视频打点信息。
* **自动场控候选**：拟定回应弹幕（如“哥哥长得明明很好看”或“T1加油”），分流至复制或自动发送模块。

### 5. 桌面 UI 层 (Presentation Layer)
* **置顶主统计浮窗**：显示聚合弹幕、新弹幕列表与置顶热词。
* **场控详情窗口**：显示梗来源、提词内容和联网依据。
* **桌宠与托盘**：提供桌面快速展开/隐藏浮窗的轻度入口。
